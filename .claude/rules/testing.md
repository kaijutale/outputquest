# テスト規約

## テストファイル配置

- コンポーネントテスト: `__tests__/` または `.test.tsx`
- ユーティリティテスト: `*.test.ts`
- E2Eテスト: `e2e/` ディレクトリ

## テスト対象

| 対象 | 優先度 | 備考 |
|------|--------|------|
| ユーティリティ関数 | 必須 | 純粋関数は特に重要 |
| カスタムフック | 推奨 | 複雑なロジックを含む場合 |
| API Routes | 推奨 | エラーハンドリングの検証 |
| Server Actions | 推奨 | データ変更処理の検証 |

## テストツール

- **Vitest / Jest**: ユニットテスト
- **React Testing Library**: コンポーネントテスト
- **MSW**: API モック
- **Playwright / Cypress**: E2Eテスト（必要時）

## テストの原則

- テストは実装の詳細ではなく、振る舞いをテストする
- モックは最小限に抑える
- 各テストは独立して実行可能にする
